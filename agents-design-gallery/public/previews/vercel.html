## 1. Visual Theme & Atmosphere

Vercel's design system is built entirely around the power of pure black and pure white — a radical commitment to chromatic neutrality that makes the brand instantly recognizable. The default dark theme places `#000000` as both the brand identity color (the iconic black triangle logo) and the page surface, creating an experience of depth through surface contrast rather than color. The palette is almost entirely achromatic, with a single electric blue accent (`#0090ff`) emerging only where interactive feedback is required. This restraint is not minimalism for its own sake — it is a statement that the product itself (running applications, deployment previews, code) is the color.

Typography is handled by Vercel's proprietary Geist typeface, a geometric sans-serif variable font spanning weights 100–900. Marketing headlines use fluid clamp-based sizing that scales from 20px on mobile to 80px on large viewports (`--text-fluid-20-80`), with tight negative letter-spacing (`-0.025em`) that reads as precision engineering, not editorial warmth. Weight 500 anchors headings while weight 400 handles body — a deliberately narrow band that avoids the heavy/thin contrast typical of fashion editorial. Geist Mono mirrors the sans at every weight level, making code-adjacent copy feel like a native sibling rather than a tool-specific afterthought.

Surface treatment is flat by design. The shadow system (`--ds-shadow-*`) uses extremely low-opacity overlays (opacity 0.06–0.06 range, `#0000000a` to `#0000000f`) to suggest elevation without visible drama. On the dark default theme, shadows become even more subtle — borders (`#ffffff25`) do the work instead. There is no glassmorphism. No `backdrop-filter: blur()` appears on the navigation; the header uses a solid `#000000` surface that becomes slightly opaque when scrolled (`--ds-background-200`).

The single most distinctive visual choice is the fluid typography scale combined with the black-on-black surface layering. Cards (`--ds-gray-100` dark = `#1a1a1a`) lift only 10 luminance points from the background (`#000000`), creating a depth that is felt rather than seen — like a matte print pressing into matte paper.

**Key Characteristics:**
- Pure black `#000000` is both brand identity and page surface — no light mode default
- Single electric blue accent `#0090ff` used only for interactive states and links on dark
- Geist variable font (100–900) as proprietary brand typeface with Geist Mono sibling
- Fluid type scale via `clamp()` — hero text scales from 20px to 80px with viewport
- Flat elevation: depth via surface contrast (`#1a1a1a` vs `#000000`), not shadow drama
- Border radius locked to 6px for all interactive components — no pill buttons in primary actions
- Pill shape (`9999px`) reserved exclusively for badges and status tags
- Shadow system uses near-invisible alphas (`#0000000a`) — functional, never decorative
- Gray scale uses 10 named stops (`--ds-gray-100` through `--ds-gray-1000`) per theme
- Accent colors (blue, green, red, amber, purple, pink, teal) live in the palette but appear only in data, badges, and status indicators

### Color Philosophy

Vercel's palette is an exercise in negative space as brand language. Status colors (red, green, amber) are part of a rich 10-stop per-hue palette (`--ds-red-100` through `--ds-red-1000`) but appear only in semantic contexts: error states, success badges, warning indicators. The effect is a palette that scales from "zero-chrome terminal" to "full-signal dashboard" without ever feeling cluttered.

## 3. Typography Rules

### Font Family

**Primary sans-serif:** Geist — Vercel's proprietary variable font, weight range 100–900, `font-display: swap`. Self-hosted via `../media/*.woff2`. Declared as `--font-sans: "Geist", "Inter", -apple-system, BlinkMacSystemFont, ...` in `:root`. The fallback chain passes through Inter and system-ui, ensuring near-identical rendering across environments.

**Monospace:** Geist Mono — variable font weight 100–900, `font-display: block`. Self-hosted. Declared as `--font-mono: "Geist Mono", Menlo, Monaco, ...`. Used for code blocks, inline code, tabular data, and terminal-style UI elements.

**Secondary sans-serif:** Space Grotesk (400, 500 via Google Fonts) — used in isolated marketing modules, not the primary copy system.

**Secondary monospace:** Space Mono (400, self-hosted via Vercel blob storage, `font-display: optional`) — used for code labels and decorative monospace callouts.

**Fallback monospace:** Roboto Mono (400, 500, 700 via Google Fonts) — declared in `--font-mono-fallback`.

**OpenType features:** No explicit `font-feature-settings` declarations found in the CSS variables. However, Geist as a variable font supports standard OpenType features. Tabular numeric contexts use Geist Mono (inherently tabular).

### Principles

- **Fluid-first display sizing**: Hero type uses `clamp()` between 1.25rem and 5rem (`--text-fluid-20-80`), allowing a single declaration to handle all viewport sizes without breakpoint patches.
- **Negative tracking at display sizes only**: Letter-spacing tightens from -0.025em at hero scale to 0em at body scale — applying negative tracking to body copy is a violation of this system.
- **Geist, not Inter, at heading weight 500**: Although Inter is in the fallback chain, Geist's geometric form at medium weight is the intended rendering. Do not swap in Inter at weight 700 for headings — Geist 500 is the brand voice.
- **Monospace is structural, not decorative**: Geist Mono appears wherever the UI implies code, terminal, or tabular data. Using Geist Mono for marketing emphasis text breaks the semantic contract.
- **Weight stays in the 400–600 band**: Vercel avoids weight 700+ for body-adjacent text. Heavy weights (700, 800) appear only in code-bold and explicit emphasis tokens.

### Whitespace Philosophy

Vercel uses whitespace as a premium signal. On a pure black page, negative space is not empty — it is the brand. Large padding between sections (48–96px) creates breathing room that implies confidence: the content is so important it needs room to stand alone. The fluid spacing tokens ensure that this generosity scales gracefully to mobile without jarring jumps, rather than aggressively compressing at `< 600px`.

The gap unit (`24px`) is the atomic building block for internal component spacing. It appears at the inter-element level (button groups, card grids, nav items) and doubles at section-to-section transitions (`48px`). This doubling rhythm makes the page feel organized without explicit visual separators.

- `0px` — Sharp corners; code blocks, terminal chrome, table cells
- `4px` — Subtle rounding; nav item hover states, small UI chips
- `6px` (`--geist-radius`) — **Default radius for all interactive components**: buttons, inputs, cards, dropdowns, modals
- `8px` (`--geist-marketing-radius`) — Marketing card rounding; slightly warmer for landing contexts
- `12px` — Medium surfaces: menus (`--material-module medium`), modal containers
- `16px` — Fullscreen overlays, large modals
- `9999px` — Pills: badges, status tags, and small callout chips only

> The Vercel DS does not use `rounded-full` or large radius (> 16px) for buttons. Pill buttons are prohibited in the core interaction layer.

## 6. Depth & Elevation

| Level | Treatment | Use |
|---|---|---|
| Flat | No shadow, `border: 1px solid #2e2e2e` | Default content areas, code blocks, table rows |
| Ambient | `0 0 0 1px #ffffff25` (via box-shadow) | Subtle card outline; replaces border-color on dark surfaces |
| Standard | `0px 1px 2px #00000029` (`--ds-shadow-xs` dark) | Input fields, small cards, inline popovers |
| Elevated | `0px 2px 2px #00000052, 0px 8px 8px -8px #00000029` (`--ds-shadow-medium` dark) | Dropdown menus, date pickers |
| Deep | `0px 1px 1px #00000005, 0px 8px 16px -4px #0000000a, 0px 24px 32px -8px #0000000f` (`--ds-shadow-2xl`) | Modals, command palette |
| Ring / Focus | `0 0 0 2px #000000, 0 0 0 4px #50a8ff` (`--ds-focus-ring` dark) | Keyboard focus indicator on interactive elements |

### Shadow Philosophy

The Vercel shadow system is intentionally invisible on the dark theme. When the page background is pure black and the card surface is `#1a1a1a`, a traditional box shadow (`0 4px 16px rgba(0,0,0,0.3)`) produces no visible contrast. Instead, depth is communicated by two signals: the 1px white-alpha border (`0 0 0 1px #ffffff25`) and the surface color step itself. Larger shadows (`--ds-shadow-modal`, `--ds-shadow-menu`) use the compound declaration that stacks the border, a micro shadow, and a spread shadow — creating perceptible lift on dark without needing opacity levels that would look muddy.

On the light theme (not the default), the same token system resolves to warm-black shadows (`#0000000a`, `#0000000f`) that are nearly invisible — continuing the flat-elevation philosophy across both modes. This system is not designed to simulate physical depth. It is designed to communicate hierarchy through clarity.

## 7. Do's and Don'ts

### Do's

- ✅ Use `Geist` variable font at weight 500 for all marketing headings — it is Vercel's proprietary typographic voice
- ✅ Use `#0090ff` (`--ds-blue-600` dark) as the sole color accent on dark surfaces — one blue, no competing hues
- ✅ Apply `border-radius: 6px` to all interactive components (buttons, inputs, cards) — the Geist radius is non-negotiable
- ✅ Use `9999px` radius only on badges and status pills — pill buttons break the flat precision aesthetic
- ✅ Set `font-display: swap` on Geist and `block` on Geist Mono — these are the declared rendering strategies in the source
- ✅ Communicate depth with surface steps (`#000000` → `#1a1a1a` → `#1f1f1f`) and the 1px white-alpha border, not shadow depth
- ✅ Scale hero type using `clamp()` between `1.25rem` and `5rem` — the fluid approach is baked into the brand's motion language
- ✅ Use `#ffffff17` (`--ds-gray-alpha-200`) for hover states on ghost and tertiary buttons on dark surfaces
- ✅ Apply negative letter-spacing only at display sizes (headings ≥ 1.25rem) — body text tracks at `0em`

### Don'ts

- ❌ Don't use `backdrop-filter: blur()` on the navigation or cards — the Vercel nav uses a solid black surface. Glassmorphism contradicts the flat-surface system.
- ❌ Don't use any color other than `#0090ff` as a CTA blue on dark surfaces — `--ds-blue-700` (`#0071f6`) is a hover variant, not an alternate primary.
- ❌ Don't use Geist weight 700 or 800 for headings — weight 500 is the marketing headline weight, 600 for section headings. Heavy weight breaks the precision tone.
- ❌ Don't apply `border-radius: 100px` or pill shapes to primary buttons — pill buttons do not exist in the Geist component system. `6px` is the button radius.
- ❌ Don't use `#50e3c2` (cyan) or `#7928ca` (violet) as brand accent colors — these are legacy/decorative tokens from the workflow gradient system, not primary brand colors.
- ❌ Don't set positive letter-spacing (`letter-spacing: 0.05em`) on headings — Vercel tracks tight (`-0.025em` to `0em`). Loose tracking reads as a different brand entirely.
- ❌ Don't use the light gray palette (`--ds-gray-100: #f2f2f2`) from the light theme on dark surfaces — the `#1a1a1a` dark variant is the correct elevated surface color.
- ❌ Don't use colored shadows (e.g., blue-tinted shadows) — the shadow system uses neutral black alphas exclusively. Colored shadows do not appear anywhere in the DS.
- ❌ Don't use white `#ffffff` as a body text color — `#ededed` (`--ds-gray-1000` dark) is the canonical text color, preserving hierarchy over pure-white decorative elements.
- ❌ Don't apply `font-family: Space Mono` to body copy — Space Mono (`font-display: optional`) is a code-label and decorative token, not a prose typeface.

|---|---|---|

| Desktop L | 1200px–1400px | Container caps at 1200px; content centered; fluid text near maximum |
| Wide | > 1400px | Container still 1200px max; hero text hits 5rem ceiling |

Primary breakpoints with high usage counts: `600px` (122 rules), `960px` (119 rules), `601px` (47 rules), `961px` (34 rules). The system uses min-width and max-width pairs, providing both mobile-first and desktop-first variants.

### Touch Targets

- Default button height: `40px` (`--height: 40px` from `.button-module__QyrFCa__button`)
- Small button height: `32px`
- Large button height: `48px`
- Minimum touch target for interactive elements: `32px` height (aligns with `--geist-space-8x`)
- Nav items on mobile: full-width tap targets via hamburger expansion

### Collapsing Strategy

From desktop to mobile, the Vercel system applies these transformations:
1. **Navigation**: Horizontal nav collapses to hamburger/drawer at `≤ 960px`. Nav item font stays at `14px`.
2. **Typography**: Hero text scales from 80px (desktop) down to 20px (mobile S) via `clamp()`. No hard breakpoint jumps.
3. **Grid**: Multi-column feature grids collapse from 3-column → 2-column → 1-column at `960px` then `600px`.
4. **Spacing**: Padding scales from `48px` section padding on desktop to `24px` on mobile.
5. **Cards**: Full-width stacking on mobile; horizontal scroll for compact card carousels.

### Image Behavior

- Framework logos and integration icons: SVG preferred; maintain aspect ratio at all sizes.
- Dashboard/product UI screenshots: served as responsive images; container clips at `border-radius: 6px` or `12px`.
- Hero product imagery: scales with viewport via `width: 100%; max-width: 1200px`.
- Gradient backgrounds: CSS-only (`linear-gradient`, `radial-gradient`) — no raster assets for backgrounds.

### Iteration Guide

1. **Always start with pure black `#000000`** as the page background — every Vercel implementation begins here. Do not use `#111111` or `#0a0a0a` as substitutes; the DS uses literal `#000000`.
2. **Geist weight 500 is the headline weight** — not 600, not 700. Apply weight 600 only for section subheadings (≤ 2rem). Never use Geist 800 on marketing copy.
3. **The 6px radius is universal** — button, input, card, tooltip, dropdown all use 6px. The only exception is badges at 9999px. Do not mix in 8px, 10px, or 12px on component edges.
4. **Depth through surface, not shadow** — when you need visual hierarchy, step the background color from `#000000` → `#1a1a1a` → `#1f1f1f`. Add `box-shadow: 0 0 0 1px #ffffff25` as the card edge. Resist adding heavy outer shadows.
