## 1. Visual Theme & Atmosphere

Stripe's visual language is a study in restrained opulence — a system that communicates premium consumer fintech through generous whitespace, razor-precise typography, and a signature purple that anchors without dominating. The palette is built on a blue-tinted neutral scale (every gray carries a cool navy undertone, from `#f8fafd` down to `#061b31`) that gives the surface a subtle coolness, distancing it from the warm grays of generic SaaS templates.

Typography is the brand's most distinctive design choice. Sohne-var operates at weight 300 for virtually all body and heading text — an unusually light weight that creates an editorial, almost magazine-quality reading experience. Weight 400 is reserved for emphasis, buttons, and interactive elements, making "bold" feel like a whisper rather than a shout. The variable font enables smooth interpolation, and `font-feature-settings: 'ss01'` activates Stripe's preferred stylistic alternates for character differentiation in data-dense contexts.

Surfaces are layered through outline strokes (`1px solid` at 50% opacity of the neutral-50 token) and soft, indigo-tinted shadows (`rgba(50,50,93,0.25)`) rather than hard elevation. Cards float with dual-shadow formulas that combine a blurred indigo ambient with a tighter black contact shadow, creating a sense of physicality without heaviness. The shadow hue deliberately echoes the brand purple — even the page depth speaks Stripe.

Gradient accents appear in the hero and product navigation as radial blurs of brand purple, magenta, and orange — expressive moments in an otherwise disciplined system. These gradients are contained to brand-immersive sections and icon treatments; they never bleed into functional UI.

**Key Characteristics:**
- Weight 300 as default text weight — editorial lightness as luxury signal
- Blue-tinted neutral scale (navy undertones from `#f8fafd` to `#061b31`)
- Indigo-hued shadows (`rgba(50,50,93,…)`) that echo the brand purple
- Tight border-radius scale (2px–6px for UI, 16px for feature cards)
- Variable font with OpenType `'ss01'` alternates enabled globally
- Restrained 4px radius on buttons — never pill-shaped
- Dashed borders as decorative section dividers (canary dashed border pattern)
- Product gradients (purple → magenta → orange) contained to hero and navigation
- 12-column grid on desktop with 16px gaps and generous page margins
- Reduced-motion support baked into navigation timing tokens

This discipline makes the brand recognizable even in grayscale: the weight-300 typography and indigo-shadow system carry the identity when color is absent.

## 3. Typography Rules

### Font Family

**Primary**: sohne-var — a variable-weight sans-serif loaded as `woff2-variations` with weight axis 1–1000. The canonical CSS value is `"sohne-var", "SF Pro Display", sans-serif` (declared via `--hds-font-family`). Two `@font-face` rules exist: one with `font-display: block` (critical text) and one with `font-display: swap` (deferred text).

**Monospace**: SourceCodePro — loaded at weight 500 (`font-display: swap`). Used for code samples, API references, and technical annotations.

**OpenType Features**: `font-feature-settings: 'ss01'` is the canonical Stripe typographic DNA — Stylistic Set 01 activates alternative character forms for improved legibility at small sizes and in data-heavy contexts.

### Principles

- **Weight 300 is the brand voice** — sohne-var at 300 creates Stripe's distinctive editorial quality. Weight 400 ("bold" in the HDS system) is reserved exclusively for interactive elements: buttons, links, and navigation labels.
- **Negative letter-spacing tightens at scale** — display sizes use `-0.02em`, sections use `-0.01em`, body uses `0em`. This progressive tracking creates visual density at large sizes without cramping smaller text.
- **Line-height compresses for display** — hero text at `1.03` is nearly solid-set, creating a billboard-like density. Body text breathes at `1.4`.
- **The weight 300/400 binary is absolute** — Stripe never uses 500, 600, or 700 on sohne-var in marketing surfaces. The entire heading-to-body hierarchy operates within a single weight step.
- **SourceCodePro at weight 500** provides just enough contrast against the surrounding light sohne-var to make code blocks visually distinct without heavy bordering.
- **`'ss01'` is non-negotiable** — disabling this feature set breaks the brand's typographic identity. It must be present on all sohne-var text.

### Shadow Philosophy

Stripe's shadow system is its most branded non-color design element. The signature dual-shadow formula pairs an indigo-tinted ambient (`rgba(50,50,93,…)`) with a neutral black contact shadow (`rgba(0,0,0,…)`). The indigo component — derived from the brand purple's warm-cool balance — ensures that even shadows feel like "Stripe." The negative spread values (`-30px`, `-18px`) pull the shadow inward, creating a floating effect without wide diffusion. This is not generic CSS elevation — it's a deliberate chromatic depth system that makes card surfaces feel like they're hovering in a subtly purple-tinted atmosphere.

## 7. Do's and Don'ts

### Do's

- ✅ Always set `font-feature-settings: 'ss01'` on all sohne-var text — this is Stripe's typographic fingerprint
- ✅ Use weight 300 for all headings and body text — weight 300 IS the brand
- ✅ Use weight 400 exclusively for interactive elements: buttons, links, navigation labels
- ✅ Apply the indigo dual-shadow formula: `rgba(50,50,93,…)` ambient + `rgba(0,0,0,…)` contact — this is how Stripe cards float
- ✅ Use the blue-tinted neutral scale (`#f8fafd` → `#061b31`) — never substitute with warm or true grays
- ✅ Keep button radius at 4px and card radius at 6px — the tight radius is a brand signal
- ✅ Use `color-mix(in srgb, #e5edf5 50%, transparent)` for card strokes — the half-opacity border is a signature treatment
- ✅ Apply `-0.02em` letter-spacing at display sizes and `-0.01em` at section sizes — progressive tightening is intentional
- ✅ Set navigation transitions to `240ms cubic-bezier(0.45,0.05,0.55,0.95)` — this is the branded easing
- ✅ Support `prefers-reduced-motion` by setting all animation durations to `0s`

### Don'ts

- ❌ Don't use weight 500, 600, or 700 on sohne-var — Stripe's entire type hierarchy operates within 300–400
- ❌ Don't use pure black `#000000` for headings or text — Stripe's darkest text is `#061b31` (deep navy)
- ❌ Don't use warm grays, true grays, or grays without blue undertone — every neutral from `#f8fafd` to `#061b31` carries cool navy
- ❌ Don't apply positive letter-spacing at any heading size — Stripe tracks tight at display, zero at body, never positive
- ❌ Don't use pill-shaped buttons (border-radius 9999px) for CTAs — primary and secondary buttons use 4px radius
- ❌ Don't substitute generic `rgba(0,0,0,0.1)` shadows for Stripe's indigo-tinted `rgba(50,50,93,…)` formula
- ❌ Don't use card radius larger than 16px — the system maxes at `--hds-space-core-radius-lg` (16px)
- ❌ Don't use SF Pro Display, Helvetica Neue, or Arial as visible fonts — sohne-var is the only font that renders; fallbacks exist only for loading states
- ❌ Don't disable `'ss01'` OpenType feature — the alternates are essential to Stripe's character shapes
- ❌ Don't use orange `#ff6118`, magenta `#f44bcc`, or ruby `#ea2261` outside of product icons and navigation gradients — these are product accents, not general-purpose colors

## 8. Responsive Behavior

### Breakpoints

| Name | Width | Key Changes |
|------|-------|-------------|
| Mobile | < 600px | 4-column grid, 0px page margin, heading-xxl shrinks to 2.125rem, hamburger nav |
| Tablet | 600px–939px | 8-column grid, 16px page margin, heading-xxl at 3rem, navigation still collapses |
| Desktop (sm) | 940px–1295px | Full navigation visible, navigation border-radius activates (6px), 8→12 column transition |
| Desktop (lg) | ≥ 1296px | 12-column grid, full content width, heading-xxl at 3.5rem, generous page margins |

### Touch Targets

Buttons maintain a minimum of 10px vertical padding (10px 16px standard), producing touch targets well above the 44px minimum. Navigation items in mobile use full-width tap targets. Interactive cards are pointer-cursor-enabled for their entire surface area.

### Collapsing Strategy

- **Typography**: heading-xxl scales from 2.125rem (mobile) → 3rem (tablet) → 3.5rem (desktop) via responsive CSS variable overrides at each breakpoint
- **Grid**: 4 → 8 → 12 columns. Content stacks vertically at mobile, shifts to 2-column at tablet, full grid at desktop
- **Navigation**: Full horizontal menu (desktop ≥ 940px) collapses to hamburger menu (< 940px) with `0.25s ease-in-out` transition for menu bars
- **Cards**: Product cards stack vertically on mobile, carousel horizontally on desktop with `var(--carousel-item-progress)` position tracking
- **Section spacing**: Vertical padding reduces proportionally — `--section-row-gap-mb`, `--section-row-gap-tb`, `--section-row-gap-dt` control per-breakpoint gaps

### Image Behavior

Dashboard and product UI images scale proportionally within their containers using `--graphic-scale` and `--graphic-aspect-ratio` tokens. Hero images use full-bleed backgrounds with contained content. Customer logos render as flat monochrome SVGs that maintain legibility at any size via the `--customerLogoColor` system.
