## 1. Visual Theme & Atmosphere

The Tiffany & Co. Press newsroom is a masterclass in luxury restraint. Where most brand newsrooms lean on dense grids and utilitarian layouts, Tiffany deploys the same discipline found in their Fifth Avenue flagships — generous whitespace, sharp edges, and a color palette so tightly controlled it reads as silence punctuated by that unmistakable robin's-egg blue.

The palette anchors on three notes: pure white surfaces, pure black typography, and the iconic Tiffany Blue (`#81d8d0`) deployed as a precise accent — never a flood. Blue appears as the logo bar background, as hover-state fills on buttons, as 3px underline reveals on navigation links, and as checkbox-checked backgrounds. It is always an event, never wallpaper. This restraint mirrors how the physical brand uses the Tiffany Blue Box — sparingly, ritually, meaningfully.

Typography follows the same luxury-editorial logic. Headlines use Sterling Display Roman, an in-house serif that carries the weight of the brand's 187-year heritage without descending into ornamentation. Body and UI text switch to Santral, a geometric sans-serif available only in Light (300) and SemiBold (600) — no middle weight, no ambiguity. The absence of a Regular (400) weight in Santral is a deliberate design choice: text is either quiet (300) or emphatic (600), never neutral.

The single most distinctive visual choice is the absolute zero-radius policy. Every button, every card, every input, every border is squared off at `0px`. No soft corners, no pill shapes. This is a Tiffany signature — the sharp edge of a platinum setting, the clean geometry of the Blue Box itself.

**Key Characteristics:**
- Tiffany Blue (`#81d8d0`) used exclusively as accent — never as background for content sections
- Zero border-radius on all interactive elements (buttons, cards, inputs)
- Dual-weight type system: Santral 300 (body) vs Santral 600 (UI/CTA) with no intermediate
- Sterling Display Roman (serif) reserved strictly for headlines
- Intentionally flat — no box-shadows, no gradients, no blur filters
- Black-and-white surface with Tiffany Blue as the only chromatic moment
- Button hover reveals Tiffany Blue background with 500ms transition — a slow, luxurious state change
- Underline animations on links grow from 0 to full width in 500ms
- 180px minimum button width — buttons are never small or timid
- WordPress on Cloudflare — bespoke theme, not a builder template

## 2. Color Palette & Roles

### Primary

- **Tiffany Blue** (`#81d8d0`): The house's registered brand color. Used for the header logo bar background, button hover fills, navigation underline accents, checkbox checked states, and keyboard focus outlines. This is not a UI utility color — it is a brand ritual.

### Brand & Dark

- **Pure Black** (`#000000`): `--wp--preset--color--black`. All body text, all heading text, button borders, button text. Black is the structural backbone of every page.
- **Dark Navy** (`#002b49`): Deep editorial navy used in immersive dark sections and feature banners.
- **Deep Navy** (`#0c3d5d`): Used in `.button.primary` variant text color — a warmer alternative to pure black for button states.

### Accent Colors

- **Tiffany Blue Light** (`#bbe7e3`): A 35% opacity tint of Tiffany Blue used for subtle background washes.
- **Tiffany Blue 50%** (`rgba(129,216,208,0.50)`): Used in focus shadows and transitional states.

### Interactive

- **Hover Fill**: `#81d8d0` — every button hover, every link underline animation, every border hover fills with Tiffany Blue
- **Focus Ring**: `1px solid #81d8d0` — keyboard focus outline
- **Focus Shadow**: `0 0 0 5px #1199ff` — secondary focus indicator for non-Tiffany elements
- **Active Opacity**: `0.6` — active button press state

### Neutral Scale

- **Black** (`#000000`): Headings, body text, button text, navigation labels
- **Dark Gray** (`#333333`): Fallback text in plugin contexts
- **Medium Gray** (`#767676`): Form labels, search labels, muted helper text
- **Gray** (`#979797`): Subtle dividers
- **Light Gray** (`#e6e6e6`): Card borders, card divider backgrounds
- **Lightest Gray** (`#f5f5f5`): Background tint for alternate sections

### Surface & Borders

- **Surface White** (`#ffffff`): Page background, card background, menu bar background
- **Border Primary** (`#e4e4e4`): Navigation borders, dropdown dividers, category link separators
- **Border Light** (`#e6e6e6`): Card borders
- **Border Medium** (`#d7d7d7`): Gallery category link borders

### Shadow Colors

The system is intentionally shadowless. No `box-shadow` values are applied to brand components — `box-shadow: none` is explicitly declared on cards and buttons. Depth is communicated exclusively through Tiffany Blue hover fills and border contrast.

### Color Philosophy

Tiffany's color discipline mirrors its retail philosophy: the Blue Box earns its power by being rare. On this surface, `#81d8d0` appears in perhaps 5% of the total pixel area — the logo bar, a hover fill, an underline animation — and that scarcity is what makes each appearance feel like an event. The rest is a monochromatic stage of black type on white ground, stripped of any competing hue. There are no warm grays, no blue-tinted backgrounds, no gradient treatments. The palette says: we are confident enough to be silent, and when we speak, we speak in Tiffany Blue.

## 3. Typography Rules

### Font Family

- **Display**: Sterling Display Roman — a proprietary serif with classical proportions, loaded from self-hosted OTF via `@font-face`. Two registrations exist (`SterlingDisplay-Roman` and `Sterling Display`) pointing to the same font file.
- **Body**: Sterling Text — a text-optimized companion serif for long-form editorial content, also self-hosted OTF.
- **UI/Sans**: Santral — a geometric sans-serif loaded in two weights only: Light (300) and SemiBold (600). The absence of Regular (400) is deliberate.
- **Icons**: FontAwesome 4.3.0 — used for search, email, and UI icons.
- **No OpenType features** are declared in the CSS. Sterling Display and Santral are used at their defaults.

### Hierarchy

| Role | Font | Size | Weight | Line Height | Letter Spacing | Features | Notes |
|------|------|------|--------|-------------|----------------|----------|-------|
| display-hero | SterlingDisplay-Roman | 42px | 400 | 1.13 | normal | — | Largest headline, reserved for hero banners |
| display-large | SterlingDisplay-Roman | 36px | 400 | 1.13 | normal | — | Section introductions |
| section-heading | SterlingDisplay-Roman | 32px | 400 | 1.13 | normal | — | Article h1, modal h2 at desktop |
| subheading-large | SterlingDisplay-Roman | 25px | 400 | 1.08 | normal | — | Article h2, card headlines |
| subheading | SterlingDisplay-Roman | 20px | 400 | 1.2 | normal | — | Intermediate heading |
| body-large | Sterling Text | 18px | 400 | 1.6 | normal | — | Editorial long-form body |
| body | Santral | 16px | 300 | 1.5 | normal | — | Standard body copy |
| body-small | Santral | 14px | 300 | 1.57 | normal | — | Sub-menu items, date/excerpt text |
| button | Santral | 14px | 600 | normal | normal | — | All buttons and CTAs |
| link | Santral | 14px | 600 | 1.4 | normal | — | CTA links, navigation items |
| caption | Santral | 12px | 300 | 1.4 | normal | — | Footer text on mobile |
| micro | Santral | 10px | 600 | 1.2 | normal | — | Search form labels |

### Principles

- Sterling Display Roman is never bolded — weight 400 is the only available weight, and adding `font-weight: bold` would trigger browser faux-bold, which would destroy the typeface's refined proportions.
- Santral operates as a binary system: 300 for reading, 600 for interaction. There is no 400 or 700 — the design system enforces a clear distinction between passive text and active UI elements.
- 14px is the dominant UI size — buttons, links, navigation, category labels, CTAs, dates, and excerpts all share this single size. Hierarchy is created through weight (300 vs 600) and typeface (Santral vs Sterling), never through size variation within the same family.
- Headlines maintain tight line-heights (1.08–1.13) to create dense, architectural headline blocks. Body text opens to 1.5–1.6 for comfortable reading.
- The `font-stretch: normal` declaration appears on every Santral usage — explicitly preventing any stretch variation.

## 5. Layout Principles

### Spacing System

Base unit: 16px (1rem). The WordPress preset spacing scale provides the foundational rhythm:
- 7px (0.44rem) — `--wp--preset--spacing--20`
- 11px (0.67rem) — `--wp--preset--spacing--30`
- 16px (1rem) — `--wp--preset--spacing--40`
- 24px (1.5rem) — `--wp--preset--spacing--50`
- 36px (2.25rem) — `--wp--preset--spacing--60`
- 54px (3.38rem) — `--wp--preset--spacing--70`
- 81px (5.06rem) — `--wp--preset--spacing--80`

The custom theme layers its own spacing on top: 30px horizontal padding on navigation, 50px on article titles, 60px–130px on single-post content columns.

### Grid & Container

- Grid container: full-width with centered Flexbox
- Maximum content width: 904px for news dropdown, wider for gallery grids
- Hero pattern: Full-bleed featured banner with side-by-side image + title panel
- Multi-column: CSS Flexbox with `.cell` units and 15px left/right padding
- Foundation-style grid underlies the layout (Zurb Foundation detected in markup patterns)

### Whitespace Philosophy

Whitespace on this surface is not empty space — it is the visual equivalent of the felt-lined interior of a Tiffany Blue Box. Article cards sit in generous margins. The navigation breathes with 30px horizontal padding per item. Single-post content indents from 60px on tablet to 130px on large screens, creating a narrow reading column that echoes editorial magazine layout. Every pixel of whitespace is deliberate: it communicates that what is presented is precious enough to deserve room.

### Border Radius Scale

- **0px**: Every brand component — buttons, cards, inputs, checkboxes, modals, images. This is a design rule, not a default.
- **50%**: Used only for circular icon containers (rare)
- **4px**: Appears only in EDD plugin defaults (non-brand third-party code)
- **9999px**: Appears once in a plugin variant (non-brand)

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat (Base) | No shadow, 1px solid border | Cards, containers, inputs, all brand components |
| Divider | 1px solid `#e4e4e4` | Navigation bar bottom, dropdown separators, category links |
| Accent | 2-3px solid `#81d8d0` | Active nav underline, hover underline animation |
| Focus Ring | `outline: 1px solid #81d8d0` | Keyboard focus on links, buttons, inputs |

The system is intentionally zero-elevation. Box-shadow is explicitly set to `none` on cards and buttons. There are no ambient shadows, no layered depth cues, no `backdrop-filter` blur. Depth is communicated exclusively through two mechanisms: border contrast (solid `#e4e4e4` dividers) and Tiffany Blue accent fills (hover states, active states). This is a conscious design decision — the luxury brand does not "lift" elements off the page because the page itself is the premium surface.

### Shadow Philosophy

Tiffany's press surface refuses shadow entirely. While the WordPress preset provides `--wp--preset--shadow--natural` and `--wp--preset--shadow--deep`, neither is used anywhere in the brand theme. Elevation in this design system is binary: an element is either on the surface (default) or highlighted by Tiffany Blue (hover/active). There is no intermediate altitude. This mirrors the physical brand — jewelry is presented on a flat velvet surface, not elevated on risers.

## 7. Do's and Don'ts

### Do's

- ✅ Use `border-radius: 0px` on every interactive element — buttons, cards, inputs, modals, image containers. Zero-radius is Tiffany's geometric signature.
- ✅ Use Santral weight 300 for body/reading text and weight 600 for interactive/CTA text — the binary weight system is intentional.
- ✅ Use Sterling Display Roman (weight 400 only) for all headlines — never apply bold or semi-bold to Sterling.
- ✅ Set button `min-width: 180px` — Tiffany buttons are generous, never cramped.
- ✅ Transition hover states at `500ms` — the slow reveal is a luxury tempo. Both `background` and `color` transition simultaneously.
- ✅ Animate link hover with a 3px-height `#81d8d0` underline growing from width 0 to 100% — this is the house's primary micro-interaction.
- ✅ Use pure black `#000000` for all text — headings, body, labels, navigation. No grays for primary text.
- ✅ Keep `box-shadow: none` on all brand components — flatness is a design decision.

### Don'ts

- ❌ Don't use border-radius above `0px` on brand components — no rounded corners, no pill buttons. The only exception is circular icon containers at `50%`.
- ❌ Don't use Santral weight 400 or 700 — these weights are not loaded. Using them would trigger browser synthesis, which destroys the typeface geometry.
- ❌ Don't apply `font-weight: bold` to Sterling Display Roman — the typeface ships only in weight 400. Faux-bold would ruin the serifs.
- ❌ Don't use any color as a surface fill except `#ffffff` (white) and `#81d8d0` (Tiffany Blue hover). No gray backgrounds, no dark sections with colored backgrounds, no gradient washes.
- ❌ Don't add `box-shadow` to cards, buttons, or containers — the design system explicitly sets `box-shadow: none`. Shadows would break the flat, editorial aesthetic.
- ❌ Don't use `backdrop-filter: blur()` — the visual language relies on solid, opaque surfaces.
- ❌ Don't create tertiary button variants with Tiffany Blue backgrounds — blue is reserved for hover states and the logo bar. At rest, buttons are white-on-black-border or black-filled.
- ❌ Don't increase the transition speed below 500ms for hover states — the slow, 500ms transition on buttons and links is a deliberate luxury tempo. Faster transitions would feel cheap.
- ❌ Don't use Tiffany Blue (`#81d8d0`) as body text color or heading color — it is an accent/interaction color only, never a text color.
- ❌ Don't use FontAwesome icons as decorative elements — they are functional UI indicators only (search, email, cart quantity).

## 8. Responsive Behavior

### Breakpoints

| Name | Width | Key Changes |
|------|-------|-------------|
| Small (mobile) | ≤ 47.99875em (~768px) | Logo bar: 63px height, logo 130px. Title padding: 30px 20px. Footer: 12px text. |
| Medium (tablet) | ≥ 48em (~768px) | Article title gets flex column layout. Sustainability chapter gaps widen to 40px. |
| Large (desktop) | ≥ 64em (~1024px) | Menu bar shows centered horizontal nav. Featured banner h2: 32px/36px. Content padding: 60px. |
| X-Large | ≥ 1440px | Single-post content padding: 130px left/right (narrow editorial column). |

### Touch Targets

- Mobile navigation: 17px top/bottom padding on menu items (total height ~48px with text)
- Buttons: minimum 180px × ~38px (10px top/bottom padding + 14px text + 4px border)
- Hamburger menu toggle: full logo bar height (63px)
- Gallery checkboxes: standard checkbox size with `#81d8d0` checked background

### Collapsing Strategy

- **Navigation**: Horizontal menu bar collapses to drilldown offcanvas panel with back-button navigation. Logo bar remains at full width with hamburger (left) and search (right).
- **Typography**: h1 drops from 32px to 30px at mobile. Modal h2 reaches 32px only at desktop (64em+).
- **Grid**: Featured gallery shifts from multi-column grid to single-column stack. Post-wrapper cells maintain 15px padding.
- **Content Width**: Single-post content column narrows progressively — 130px padding at 1440px+, 60px at desktop, 30px at tablet, 20px at mobile.
- **Footer**: Typography drops to 12px. Headings switch to Santral 14px 600 matching body.

### Image Behavior

- Featured post images: `object-fit: cover` with ratio-box aspect ratio containers (`padding-top: 61.39%` for wide format)
- Gallery images: minimum 1px height, scaled to container width
- Logo: responsive from 130px (mobile) to 160px (desktop) max-width

### Iteration Guide

1. Always declare `border-radius: 0` explicitly on buttons, cards, and inputs — browser defaults will add rounding otherwise. Zero-radius is the single strongest visual identifier of this design system.
2. Load only Santral 300 and 600 — if you need an intermediate weight, you don't. The binary system is the brand. Use 300 for passive text, 600 for anything clickable.
3. Sterling Display Roman is headline-only and weight 400-only. Never set `font-weight: bold` on Sterling text. Never use Sterling for body copy — that's what Sterling Text is for.
4. Every hover interaction fills with `#81d8d0` over 500ms. The slow transition is non-negotiable — it's the tempo that separates luxury from utility. Apply `transition: background 0.5s, color 0.5s`.
5. The underline animation pattern repeats everywhere: an `::after` pseudo-element with `height: 3px`, `background: #81d8d0`, `width: 0`, `transition: width 0.5s`, growing to full width on `:hover`. Implement this on all text CTAs and nav links.
6. Buttons must be 180px minimum width with 10px vertical / 40-50px horizontal padding. Small, tight buttons are off-brand. White-outlined for default, black-filled for primary actions.
7. Use `#000000` for ALL text — headings, body, labels, placeholders. The only exception is `#767676` for form labels and `#ffffff` for white-on-black button text. Never use dark gray approximations like `#333` or `#1d1d1f` for brand text.
8. The page is flat. No shadows, no gradients, no glassmorphism. If you catch yourself adding `box-shadow`, remove it. Depth is communicated through Tiffany Blue accent lines and border contrast only.
