## 1. Visual Theme & Atmosphere

Aman's digital presence embodies the same unhurried restraint that defines its physical properties. The palette is anchored in a warm sand cream (`#f3eee7`) — not white, not gray, but a tone that suggests raw linen, sun-warmed stone, handmade paper. Against this foundation, near-black (`#313131`) and true black (`#000000`) provide the only contrast needed. There are no accent colors fighting for attention. No gradient hero. No electric blue CTA. The brand's single most distinctive visual choice is this radical absence — the confidence to let a warm monochrome palette and full-bleed photography do all the work.

Typography reinforces the editorial luxury positioning through Lyon Display Web at weight 300 — a serif so light it barely touches the page. Headlines read like a whisper, not a shout. The sans-serif companion, Whitney SSm, handles functional elements (buttons, navigation, labels) with the same lightness at weight 300-400. This High-Low pairing — editorial serif for display, humanist sans for function — creates the tension between contemplative and operational that defines the Aman guest experience.

Surfaces are uniformly flat. There is no glassmorphism, no layered card elevation, no frosted blur. Depth is communicated through photography alone — full-bleed images that break the layout grid, drawing the eye through the page like walking through an Aman property where each turn reveals a new vista. The UI deliberately recedes. Buttons are borderless rectangles without radius. Cards are frameless. Navigation is small, tracked, and uppercase. The interface never competes with the imagery it frames.

Motion, when present, is slow and deliberate — 0.4s transitions with ease-out curves. No bounce, no spring, no elastic. The site respects `prefers-reduced-motion` throughout, reflecting the brand philosophy that luxury is the absence of friction.

**Key Characteristics:**
- Warm sand cream (`#f3eee7`) as primary surface, not white — this IS the brand
- Lyon Display Web at weight 300 for all serif headlines — barely-there lightness
- Whitney SSm for all functional text — humanist sans with editorial poise
- Zero border-radius on all interactive elements — sharp rectangles only
- Full-bleed photography dominance — UI recedes completely
- Flat surfaces with no shadows, no glass, no elevation hierarchy
- Radical color restraint — monochrome warm palette with no accent color
- Slow motion (0.4s minimum transitions) with ease-out curves
- Uppercase tracked labels and subtitles at 0.722rem
- Custom Drupal CMS running Aman's bespoke theme (not a builder platform)

## 5. Layout Principles

### Spacing System

Base unit is 14px (the `:root` font-size). The spacing scale uses fractional rem values reflecting a 14px base:

- 0.25rem (3.5px) — inner padding for icon buttons
- 0.5rem (7px) — tight gaps
- 0.71rem (10px) — card padding
- 1rem (14px) — standard unit
- 1.14rem (16px) — body text leading
- 1.75rem (24.5px) — section gaps
- 2.14rem (30px) — generous gaps
- 3.07rem (43px) — property nav height
- 4.07rem (57px) — major section padding
- `clamp(1.75rem, 3.5vw, 3.5rem)` — fluid section padding

### Grid & Container

The layout uses a flexible grid system without a fixed container max-width visible in the extracted CSS. Content areas appear to use percentage-based widths (8.3% / 17.5% gutters suggest a 12-column grid with generous outer margins). Full-bleed imagery breaks the grid at multiple points per page.

Key layout patterns:
- Destinations grid: 3-column grid with 2rem column gap
- Card advanced layout: 33.33% width for text column + remaining for media
- Horizontal cards: Flexbox with 1.75rem margin between image and copy
- Hero sections: Full-viewport imagery with overlaid text

### Whitespace Philosophy

Aman uses whitespace as the primary luxury signal. Section padding in the `clamp(1.75rem, 3.5vw, 3.5rem)` range ensures generous breathing room that scales responsively. Content density is deliberately low — fewer elements per viewport, more room for each to exist. The wide button padding (0.8em 4em — nearly 5x wider than tall) is a whitespace statement: even within interactive elements, the brand asserts that space is abundant. This treatment mirrors the brand's physical properties where architecture frames absence as much as presence.

### Border Radius Scale

- **0px**: Buttons, cards, inputs, images, badges — everything. This is the luxury design grammar.
- **2px**: Appears only in administrative/CMS UI contexts
- **4px**: Administrative elements (Drupal admin theme fallbacks)
- **7px**: Isolated utility contexts
- **50%**: Only for circular icon buttons (close, navigation arrows)

The brand radius is 0. Period. Rounded corners are a SaaS tell that Aman rigorously avoids.

### Shadow Philosophy

The system is intentionally low-elevation. Aman communicates depth exclusively through surface color contrast (sand vs. slightly-warmer-sand for cards) and photography (which carries all the spatial depth the eye needs). There are no layered card shadows, no blur-glass panels, no ambient glow. The single shadow declaration found is a Drupal admin-theme artifact, not a brand choice. When building for Aman, depth comes from composition and imagery, never from `box-shadow`.

## 7. Do's and Don'ts

### Do's

- **Do** use Lyon Display Web at weight 300 for all display headlines — this featherweight serif IS the Aman voice
- **Do** use `#f3eee7` (Aman Sand) as the primary surface, not `#ffffff` — the warm cream is the single most important brand token
- **Do** use zero border-radius on all buttons, cards, and inputs — sharp rectangles are a deliberate luxury code
- **Do** use `#f3eee7` (Aman Cream) as button text on dark backgrounds, not pure `#ffffff` — this warmth shift is intentional
- **Do** use Whitney SSm for all functional text (buttons, nav, labels, forms) — the serif/sans boundary separates editorial from interface
- **Do** use 0.4s minimum transition durations with ease-out curves — honor the `prefers-reduced-motion` query
- **Do** use uppercase tracked text (letter-spacing 0.1-0.15rem) for subtitles and labels — this is the primary metadata motif
- **Do** let photography dominate — the UI frames, never competes
- **Do** use wide button padding (0.8em 4em) — the horizontal generosity is a luxury signal

### Don'ts

- **Don't** use pure white (`#ffffff`) as a page background — Aman's surface is `#f3eee7`, and the entire palette is calibrated to this warmth
- **Don't** use pure white (`#ffffff`) as text on dark buttons — use Aman Cream `#f3eee7` instead to maintain the warm palette consistency
- **Don't** add any border-radius to buttons, cards, or images — rounded corners at any value (even 4px) signal SaaS, not luxury hospitality
- **Don't** use bold weight (600-700) for Lyon Display Web headlines — weight 300 is the brand voice; bold serif reads editorial-magazine, not Aman's contemplative whisper
- **Don't** add drop-shadows or box-shadows to cards — Aman's depth language is flat surface differentiation plus photography, not elevation
- **Don't** add backdrop-filter blur to any element — the visual language relies on solid warm surfaces, not glass
- **Don't** use bounce, spring, or elastic easing on any animation — these read playful/tech, not luxury hospitality
- **Don't** use geometric sans-serif (Inter, Helvetica Neue, SF Pro) for display headlines — only Lyon Display Web carries the editorial serif voice
- **Don't** introduce chromatic accent colors (blue, green, red) for CTAs — Aman operates in monochrome warm neutrals exclusively
- **Don't** use icon-heavy navigation or emoji — Aman uses text and photography, never iconography as primary navigation
- **Don't** create pricing tier cards, feature comparison grids, or dashboard preview screenshots — these are SaaS patterns incompatible with Aman's experiential positioning
- **Don't** set `:root { font-size: 16px }` — Aman's base is 14px, and all rem values depend on this base
