## 1. Visual Theme & Atmosphere

Aman's digital presence embodies the same unhurried restraint that defines its physical properties. The palette is anchored in a warm sand cream (`#f3eee7`) — not white, not gray, but a tone that suggests raw linen, sun-warmed stone, handmade paper. Against this foundation, near-black (`#313131`) and true black (`#000000`) provide the only contrast needed. There are no accent colors fighting for attention. No gradient hero. No electric blue CTA. The brand's single most distinctive visual choice is this radical absence — the confidence to let a warm monochrome palette and full-bleed photography do all the work.

Typography reinforces the editorial luxury positioning through Lyon Display Web at weight 300 — a serif so light it barely touches the page. Headlines read like a whisper, not a shout. The sans-serif companion, Whitney SSm, handles functional elements (buttons, navigation, labels) with the same lightness at weight 300-400. This High-Low pairing — editorial serif for display, humanist sans for function — creates the tension between contemplative and operational that defines the Aman guest experience.

Surfaces are uniformly flat. There is no glassmorphism, no layered card elevation, no frosted blur. Depth is communicated through photography alone — full-bleed images that break the layout grid, drawing the eye through the page like walking through an Aman property where each turn reveals a new vista. The UI deliberately recedes. Buttons are borderless rectangles without radius. Cards are frameless. Navigation is small, tracked, and uppercase. The interface never competes with the imagery it frames.

Motion, when present, is slow and deliberate — 0.4s transitions with ease-out curves. No bounce, no spring, no elastic. The site respects `prefers-reduced-motion` throughout, reflecting the brand philosophy that luxury is the absence of friction.

**Key Characteristics:**
- Warm sand cream (`#f3eee7`) as primary surface, not white — this IS the brand
- Lyon Display Web at weight 300 for all serif headlines — barely-there lightness
- Whitney SSm for all functional text — humanist sans with editorial poise
- Zero border-radius on all interactive elements — sharp rectangles only
- Full-bleed photography dominance — UI recedes completely
- Flat surfaces with no shadows, no glass, no elevation hierarchy
- Radical color restraint — monochrome warm palette with no accent color
- Slow motion (0.4s minimum transitions) with ease-out curves
- Uppercase tracked labels and subtitles at 0.722rem
- Custom Drupal CMS running Aman's bespoke theme (not a builder platform)

## 2. Color Palette & Roles

### Primary

- **Aman Black** (`#000000`): Brand identity color. Appears in the logotype, primary CTA fills (button--black), border on outlined buttons, and hero overlay text. Aman's identity IS black-on-sand.

### Brand & Dark

- **Charcoal Dark** (`#313131`): Primary body text, navigation headings, dark button variant fills, booking bar buttons. The warm near-black that replaces pure black for readable text.
- **Charcoal** (`#404040`): Button hover states for black variant, secondary interactive element backgrounds. Slightly lighter than body text for state differentiation.
- **Stone** (`#585858`): Charcoal button fills, footer link text, tertiary interactive backgrounds. The mid-tone in Aman's grayscale.

### Accent Colors

Aman intentionally has no chromatic accent color. The entire palette operates within warm neutrals. The closest to an "accent" is the contrast between black elements and the sand surface — this tension IS the visual accent. Subdued warm tones like `#b38b5b` (warm gold) and `#caa48d` (terra) appear in extremely limited decorative contexts but are not systematically deployed.

### Interactive

- **Hover on Black Button**: `#404040` background with `#ffffff` text
- **Hover on Outlined Button**: `#ffffff` background with `#000000` text
- **Hover on Charcoal Button**: `#585858` background with `#ffffff` text
- **Hover on Dark Button**: `#625f5f` background with `#f3eee7` text
- **Focus Ring**: `inset 0 0 0 1px #e6e6e6` box-shadow (inset border, not outline)
- **Active State**: Same as focus — `#585858` bg with inset `#e6e6e6` ring
- **Disabled**: `#e6e6e6` text and border (flattened, no contrast)

### Neutral Scale

- **Near-Black** (`#313131`): Primary text
- **Dark Gray** (`#404040`): Secondary emphasis, hover states
- **Stone** (`#585858`): Tertiary text, footer elements
- **Warm Gray** (`#82847f`): Muted text, placeholders, metadata
- **Stone Light** (`#aaa6a3`): Light borders, button--light border
- **Sand Border** (`#dad9d7`): Default dividers and borders (33 uses)
- **Pale Sand** (`#e6e2db`): Alternate surface, dark-sand variant
- **Sand** (`#f3eee7`): Primary surface (44 uses)
- **Light Parchment** (`#f2f0ed`): Card backgrounds, form field backgrounds

### Surface & Borders

- **Primary Divider** (`#dad9d7`): Most common border color across the site. Warm-toned, not gray.
- **Secondary Divider** (`#e6e6e6`): Focus rings, disabled states. Cooler than the primary divider.
- **Accent Border** (`#d5d1c8`): Stronger warm border for emphasized dividers.
- **Subtle Border** (`#aaa6a3`): Light button variant border.

### Shadow Colors

The system is intentionally shadowless. The single shadow declaration found (`0.07rem 0.07rem 0.43rem rgba(0,0,0,0.1)`) appears on one utility button and is not part of the brand language. Depth is communicated entirely through surface color contrast and photography.

### Color Philosophy

Aman's palette is an exercise in restraint that mirrors the brand's physical philosophy — luxury is what you remove, not what you add. The warm sand cream (`#f3eee7`) surface is the most important color decision on the entire site. It rejects the default white of the web, establishing an immediate material quality — linen, stone, paper. Every other color exists in service of this foundation. Text is warm near-black, not pure black. Borders are warm gray, not cool gray. Even the "white" hover state on outlined buttons reads differently against the sand surface than it would on pure white. This thermal consistency — everything slightly warm — is what makes the palette feel intentional rather than default.

## 3. Typography Rules

### Font Family

**Primary Serif:** Lyon Display Web (light weight 300) — the display voice. Used for all headlines, body text, and editorial content. Self-hosted from Aman's custom theme at `/themes/custom/aman/assets/dist/font/LyonDisplay-Light-Web.woff2`.

**Body Serif:** Lyon Text Web (regular weight 400) and Lyon (custom Aman cut, regular + semibold) — for running body text when a slightly heavier read is needed.

**Primary Sans:** Whitney SSm (weights 300, 400; normal + italic) — the functional voice. Used for buttons, navigation, labels, subtitles, links, and all UI chrome. Self-hosted.

**Monospace:** Source Code Pro (weights 200-900) — for code contexts. Full weight range available.

**Restaurant Specialty Fonts:** Calligraph810 (calligraphic script for Arva restaurant), SuisseBP (Swiss grotesque for Nama restaurant), Beausite (slick geometric for Nura restaurant). These override Lyon Display Web within restaurant-specific page themes.

**CJK Support:** NotoSerifJP and NotoSerifSC (weights 200, 400) for Japanese and Simplified Chinese locales.

No OpenType feature flags are explicitly enabled in the CSS.

### Principles

- Lyon Display Web at weight 300 is Aman's typographic DNA — this featherweight serif carries the entire editorial voice. Never use weight 400+ for Lyon Display headlines.
- Whitney SSm is the operational typeface — every element the user touches (buttons, links, nav, forms) speaks in Whitney. The serif/sans boundary is the boundary between editorial content and functional interface.
- The base font-size is 14px (`:root { font-size: 14px }`), making all rem values relative to 14. A headline at `2.29rem` resolves to 32px, not 36.6px.
- Subtitle/label text is always uppercase with generous tracking (0.1-0.15rem). This tracked-uppercase pattern marks metadata and categorical content.
- Letter-spacing on buttons is exactly `0.05rem` — tight enough to read at small sizes, wide enough to breathe on the sand background.
- Aman never uses bold weight for headlines. The heaviest weight in the display hierarchy is 300. Boldness is reserved for functional emphasis in forms (weight 400 or 500 for Lyon Text Semibold).

## 5. Layout Principles

### Spacing System

Base unit is 14px (the `:root` font-size). The spacing scale uses fractional rem values reflecting a 14px base:

- 0.25rem (3.5px) — inner padding for icon buttons
- 0.5rem (7px) — tight gaps
- 0.71rem (10px) — card padding
- 1rem (14px) — standard unit
- 1.14rem (16px) — body text leading
- 1.75rem (24.5px) — section gaps
- 2.14rem (30px) — generous gaps
- 3.07rem (43px) — property nav height
- 4.07rem (57px) — major section padding
- `clamp(1.75rem, 3.5vw, 3.5rem)` — fluid section padding

### Grid & Container

The layout uses a flexible grid system without a fixed container max-width visible in the extracted CSS. Content areas appear to use percentage-based widths (8.3% / 17.5% gutters suggest a 12-column grid with generous outer margins). Full-bleed imagery breaks the grid at multiple points per page.

Key layout patterns:
- Destinations grid: 3-column grid with 2rem column gap
- Card advanced layout: 33.33% width for text column + remaining for media
- Horizontal cards: Flexbox with 1.75rem margin between image and copy
- Hero sections: Full-viewport imagery with overlaid text

### Whitespace Philosophy

Aman uses whitespace as the primary luxury signal. Section padding in the `clamp(1.75rem, 3.5vw, 3.5rem)` range ensures generous breathing room that scales responsively. Content density is deliberately low — fewer elements per viewport, more room for each to exist. The wide button padding (0.8em 4em — nearly 5x wider than tall) is a whitespace statement: even within interactive elements, the brand asserts that space is abundant. This treatment mirrors the brand's physical properties where architecture frames absence as much as presence.

### Border Radius Scale

- **0px**: Buttons, cards, inputs, images, badges — everything. This is the luxury design grammar.
- **2px**: Appears only in administrative/CMS UI contexts
- **4px**: Administrative elements (Drupal admin theme fallbacks)
- **7px**: Isolated utility contexts
- **50%**: Only for circular icon buttons (close, navigation arrows)

The brand radius is 0. Period. Rounded corners are a SaaS tell that Aman rigorously avoids.

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat | No shadow, no border | Cards, main content areas, hero overlays |
| Hairline | 1px solid `#dad9d7` | Dividers, form field underlines, button outlines |
| Inset Ring | `inset 0 0 0 1px #e6e6e6` | Focus/active states on dark buttons |
| Overlay | `rgba(0,0,0,0.7)` full-screen | Navigation overlay, mobile menu backdrop |
| Content Shadow | `0.07rem 0.07rem 0.43rem rgba(0,0,0,0.1)` | Single utility instance — not a systematic pattern |

### Shadow Philosophy

The system is intentionally low-elevation. Aman communicates depth exclusively through surface color contrast (sand vs. slightly-warmer-sand for cards) and photography (which carries all the spatial depth the eye needs). There are no layered card shadows, no blur-glass panels, no ambient glow. The single shadow declaration found is a Drupal admin-theme artifact, not a brand choice. When building for Aman, depth comes from composition and imagery, never from `box-shadow`.

## 7. Do's and Don'ts

### Do's

- **Do** use Lyon Display Web at weight 300 for all display headlines — this featherweight serif IS the Aman voice
- **Do** use `#f3eee7` (Aman Sand) as the primary surface, not `#ffffff` — the warm cream is the single most important brand token
- **Do** use zero border-radius on all buttons, cards, and inputs — sharp rectangles are a deliberate luxury code
- **Do** use `#f3eee7` (Aman Cream) as button text on dark backgrounds, not pure `#ffffff` — this warmth shift is intentional
- **Do** use Whitney SSm for all functional text (buttons, nav, labels, forms) — the serif/sans boundary separates editorial from interface
- **Do** use 0.4s minimum transition durations with ease-out curves — honor the `prefers-reduced-motion` query
- **Do** use uppercase tracked text (letter-spacing 0.1-0.15rem) for subtitles and labels — this is the primary metadata motif
- **Do** let photography dominate — the UI frames, never competes
- **Do** use wide button padding (0.8em 4em) — the horizontal generosity is a luxury signal

### Don'ts

- **Don't** use pure white (`#ffffff`) as a page background — Aman's surface is `#f3eee7`, and the entire palette is calibrated to this warmth
- **Don't** use pure white (`#ffffff`) as text on dark buttons — use Aman Cream `#f3eee7` instead to maintain the warm palette consistency
- **Don't** add any border-radius to buttons, cards, or images — rounded corners at any value (even 4px) signal SaaS, not luxury hospitality
- **Don't** use bold weight (600-700) for Lyon Display Web headlines — weight 300 is the brand voice; bold serif reads editorial-magazine, not Aman's contemplative whisper
- **Don't** add drop-shadows or box-shadows to cards — Aman's depth language is flat surface differentiation plus photography, not elevation
- **Don't** add backdrop-filter blur to any element — the visual language relies on solid warm surfaces, not glass
- **Don't** use bounce, spring, or elastic easing on any animation — these read playful/tech, not luxury hospitality
- **Don't** use geometric sans-serif (Inter, Helvetica Neue, SF Pro) for display headlines — only Lyon Display Web carries the editorial serif voice
- **Don't** introduce chromatic accent colors (blue, green, red) for CTAs — Aman operates in monochrome warm neutrals exclusively
- **Don't** use icon-heavy navigation or emoji — Aman uses text and photography, never iconography as primary navigation
- **Don't** create pricing tier cards, feature comparison grids, or dashboard preview screenshots — these are SaaS patterns incompatible with Aman's experiential positioning
- **Don't** set `:root { font-size: 16px }` — Aman's base is 14px, and all rem values depend on this base

### Iteration Guide

1. **Surface color is `#f3eee7`, not white.** Every background — page, header, cards, forms — uses a warm sand tone. If anything looks "too digital," check if you accidentally used `#ffffff`.
2. **Lyon Display Web weight 300 only for headlines.** If a headline looks heavy, you're using weight 400+. The featherweight serif is the single strongest brand signal.
3. **Zero border-radius everywhere.** Before shipping any component, check that buttons, cards, inputs, and images all have `border-radius: 0`. Even 2px will break the luxury code.
4. **Button text on dark fills is `#f3eee7`, not `#ffffff`.** This warm cream-on-black pairing is deliberate. Pure white reads too cold against Aman's warm palette.
5. **Button padding is wide: 0.8em 4em.** This 1:5 vertical-to-horizontal ratio is intentional luxury spacing. Slim variant uses 0.6em 2.7em. Never compress to 0.5em 1em.
6. **Transitions are 0.4s with ease-out.** Wrap all interactive transitions in `@media (prefers-reduced-motion: no-preference)` and use at minimum 0.4s duration. No bounce, no spring.
7. **Font base is 14px, not 16px.** All rem calculations must account for `:root { font-size: 14px }`. A `2rem` value resolves to 28px, not 32px.
8. **WhitneySSm for function, Lyon for content.** If you're building a button, nav, form, or label — use WhitneySSm. If you're displaying editorial content — use Lyon Display Web. This boundary is sacrosanct.

## 12. Motion Deep Dive

### Signature Tempo: 4 seconds
The most-used duration is 4s (51 occurrences), used for hero carousel crossfades. This is 5-10x slower than typical web transitions and is a deliberate brand choice — it forces the viewer to slow down, matching the physical experience of arriving at an Aman property.

### Easing Curves
```css
/* Primary — ExpoOut: fast entrance, very slow settle */
cubic-bezier(0.19, 1, 0.22, 1)

/* Secondary — similar exponential out */
cubic-bezier(0.23, 1, 0.32, 1)
```

Both curves create a "glide into place" effect — objects arrive quickly but settle slowly, like a heavy door closing on hydraulic hinges.
