## 1. Visual Theme & Atmosphere

Hermès Paris is the world's most exclusive luxury house, and its digital presence channels that heritage with disciplined restraint. The palette is built on warm neutrals — a parchment-ivory ground (`#fcf7f1`) that recalls aged vellum, paired with pure black text and UI elements that float on this warm canvas with absolute precision. There is no blue, no gradient, no gloss. The entire surface treatment reads as physical: paper, leather, silk.

Typography is the centerpiece of the Hermès digital identity. Two typefaces carry the entire system: **Manrope** (a geometric sans-serif variable font, 200–800) for all functional UI — navigation, body text, buttons, labels — and **EB Garamond** (a classical serif variable font, 400–800, with italic) for editorial and storytelling moments. A third face, **Overpass Mono** (variable, 300–700), serves as the secondary system font for headings and labels, introducing a monospaced cadence that subtly nods to the craft of typesetting. The heading hierarchy uses Overpass Mono at weight 300 in **all-caps** — an unconventional choice that gives the page the air of a printed exhibition catalogue rather than a website.

Surfaces are intentionally flat. There are no layered shadows, no glassmorphism, no backdrop-blur. Depth is communicated exclusively through border lines and surface color shifts (warm parchment → slightly cooler cream → white). This is a brand that trusts whitespace and typography to create hierarchy, never decoration.

The single most distinctive visual choice is the **warm parchment background** (`#fcf7f1` / `#f6f1eb`) paired with monospaced uppercase headings at weight 300. This combination is immediately recognizable and cannot be mistaken for any other luxury e-commerce site.

**Key Characteristics:**
- Warm parchment-ivory backgrounds (`#fcf7f1`, `#f6f1eb`) — never pure white, never cool gray
- Pure black (`#000000`) as the sole UI color — buttons, text, borders, focus rings
- All headings in Overpass Mono weight 300, uppercase, zero letter-spacing
- EB Garamond italic for editorial/storytelling moments — the brand's voice of intimacy
- Completely flat surfaces — zero shadows, zero elevation, zero blur
- Buttons are sharp-cornered (0px default) or gently rounded (4px) — never pill-shaped
- Navigation font at 10px (0.625rem) weight 700 with 1px letter-spacing — restrained and authoritative
- No accent color in UI — Hermès Orange lives in the logo and bags, not in the interface
- Transparency and transition easing via `cubic-bezier(.25,.8,.25,1)` for overlays

## 2. Color Palette & Roles

### Primary

- **Hermès Orange** (`#f6811e`): The iconic brand identity color. Appears on the physical Hermès box, shopping bag, and brand mark. On the digital surface, it is used **with extreme restraint** — the logo and select brand moments only. It never appears as a CTA, link, or UI accent. This deliberate absence IS the digital brand strategy.

### Brand & Dark

- **Pure Black** (`#000000`): The dominant UI color. Used for body text, headings, button fills, borders, focus rings, and all interactive elements. Black is the brand's digital authority.
- **White** (`#ffffff`): Used for text on dark surfaces, tertiary button text/border, and content card surfaces.

### Neutral Scale

- **Slate** (`#696969`): Label text, secondary/muted information, form field labels.
- **Silver Disabled** (`#919191`): Disabled button text, disabled border color, tertiary UI elements.
- **Mid Gray** (`#444444`): Occasional intermediate text weight.
- **Light Silver** (`#cbcbcb`): Hover state for inner box-shadows on alerts.

### Surface & Borders

- **Hermès Ivory** (`#fcf7f1`): Primary page background and header. The warmest of the three surface tones.
- **Hermès Parchment** (`#f6f1eb`): Secondary surface background (h-root body), notification overlays, footer link borders. Slightly deeper warmth.
- **Hermès Cream** (`#fffcf7`): Footer background. The lightest warm tone — almost white with a yellow cast.
- **Warm White** (`#f5f5f5`): Modal backgrounds. The system's only neutral-gray surface.

### Interactive

- **Focus Ring Black** (`#000000`): 2px solid outline on focus-visible, 2px offset — used on light backgrounds.
- **Focus Ring White** (`#ffffff`): 2px solid outline on focus-visible — used on dark/editorial backgrounds.

### Shadow Colors

This system is intentionally shadowless. The only box-shadow usage is `inset 0 0 6px 6px #000` on alert/notification components — a thick inset ring rather than a lifted elevation.

### Color Philosophy

The Hermès digital palette is an act of radical restraint. Where most luxury brands reach for dark mode drama or gradient opulence, Hermès deploys warm parchment tones that evoke the tactile experience of their packaging — the cardboard box, the tissue paper, the ribbon. Black does all the work of hierarchy and interaction. The iconic orange is intentionally withheld from the UI, preserving it as a purely physical and brand-mark experience. This absence makes the orange MORE powerful — it lives in memory, not on screen.

## 3. Typography Rules

### Font Family

- **Primary**: Manrope (variable, 200–800, sans-serif) — all functional UI, body text, buttons, navigation labels
- **Editorial**: EB Garamond (variable, 400–800, roman + italic, serif) — storytelling headings, editorial immersive sections, display moments
- **Secondary**: Overpass Mono (variable, 300–700, monospace) — heading hierarchy (H1–H5), labels, category navigation
- **Decorative**: Filosofia, Akkurat, Jungle-Regular, BridesdeGala-Regular, Mors-Regular-Auto — declared as CSS vars but reserved for specific seasonal/editorial campaigns; zero references in the base system

### Principles

- Overpass Mono headings at weight 300 in uppercase are the signature Hermès digital voice — they read like gallery placards, not web headings
- EB Garamond italic is reserved exclusively for editorial storytelling — never used for UI, navigation, or functional copy
- Base font-size is set to 75% (12px) on the body, making all rem calculations relative to this reduced baseline — respect this when computing pixel equivalents
- Letter-spacing is 0 across the entire system EXCEPT for navigation labels (1px) and CTA labels (1px) — spaced type is a signal of interactive affordance
- Weight 300 dominates headings and modals; weight 400 is the body default; weight 500 is for labels; weight 700 is restricted to navigation only
- All heading text is `text-transform: uppercase` — Hermès never uses title case or sentence case for section headings

### Cards & Containers

The Hermès system does not define a traditional "card" component with borders and shadows. Product tiles and editorial blocks sit directly on the warm parchment background without borders, shadows, or explicit card containers. Content separation is achieved through generous whitespace and typographic hierarchy alone. If your context requires a card, use a white (`#ffffff`) background on the warm parchment surface with 0px radius and no border — the surface contrast alone provides the visual separation.

### Inputs & Forms

**Text Input** (`input-text`)
- Background: transparent
- Text: `#000000`
- Border: bottom only — 1px solid `#000000`
- Radius: 0px
- Padding: 0px
- Focus: 1px solid `#000000` bottom border (same as default — the border becomes visible via text-indent transition)
- The search input uses a sliding text-indent animation (`.3s ease-out`) to reveal the input field
- This is a marketing/e-commerce surface — form labels inherit from `body-small`, helper text from `caption`

### Badges / Tags / Pills

The system uses minimal badge patterns. Notification banners use `#f6f1eb` or `#f5f5f5` backgrounds. No pill-shaped badges are present. Category labels use the `label-small` typography pattern (Overpass Mono, 10px, weight 500, uppercase) without explicit badge containers.

### Navigation

**Header Navigation** (`nav-header`)
- Background: `#fcf7f1` (Hermès Ivory)
- Text: `#000000`
- Height: 64px desktop, 50px mobile (with 110px for menu-expanded state)
- Transparency: supports transparent mode (`#fcf7f1ba` — 73% opacity) for hero overlays
- Transition: `transform .3s` for scroll-based show/hide
- No backdrop-filter — the header uses a semi-transparent warm tone, not blur
- Navigation links: 10px Manrope weight 700, uppercase, 1px letter-spacing

**Sub-navigation menu**
- Height: 46px (`--header-menu-height`)
- Separator: 1px × 11px `#f6f1eb` vertical divider between menu items

### Decorative Elements

- **Underline links**: Editorial links on dark backgrounds use `border-bottom: 1px solid #fff` instead of `text-decoration: underline`, with uppercase text at 11px (body-default) and 1px letter-spacing
- **Inset ring**: Alert/notification hover uses `box-shadow: inset 0 0 6px 6px #000` — a thick inset ring rather than elevation shadow
- **Progress bar**: 2px height, bottom-right and top-right border-radius of 1px — an almost invisible loading indicator
- **Footer separator**: `border-bottom: 1px solid #f6f1eb` between footer link groups — the only visible horizontal rule in the system

## 5. Layout Principles

### Spacing System

Base unit: 8px. The system uses a spacing scale of 4, 8, 12, 15, 16, 20, 24, 32, 48px. The 15px value is notable — it appears frequently in horizontal padding (`0 15px`, `16px 15px 8px`) and reflects a deliberate asymmetry in the vertical/horizontal rhythm. Side padding on mobile is consistently 15px (0.9375rem).

### Grid & Container

- Max content width: 1290px with `margin: auto` and `padding: 0 15px` (0.9375rem) on desktop (≥1024px)
- Full-width sections: header, hero, editorial immersive blocks
- No explicit 12-column grid is declared — Hermès uses a flexible layout system with percentage-based widths
- Product grids adapt from 2-column on mobile to 4-column on desktop

### Whitespace Philosophy

Hermès treats whitespace as a luxury material. The generous padding values (30px, 40px, 72px top padding on some sections) and the restrained content density create a digital experience that mirrors walking through a Hermès boutique — few items, carefully placed, with breathing room that communicates exclusivity. The parchment background color turns negative space into positive brand expression.

### Border Radius Scale

- **0px** — Default for inputs, most buttons, cards, all surfaces
- **4px** — Buttons with `.border-radius-default`, select group borders
- **10px** — Rare decorative radius
- **50px** — Rounded UI elements (pill-like, rare)
- **50%** / **100%** — Circular elements (close buttons, avatars, ripple effects)

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat (default) | No shadow, surface color contrast only | All cards, containers, product tiles |
| Inset Ring | `inset 0 0 6px 6px #000` | Alert/notification hover state |
| Overlay Dark | `background: #00000080` (50% opacity) | Modal backdrop |
| Overlay Light | `background: #ffffffe6` (90% opacity) | Notification overlay |
| Overlay Mid | `background: #0000004d` (30% opacity) | Secondary overlay |
| Focus Ring | `outline: 2px solid #000` / `2px solid #fff` | Keyboard focus indicator |

### Shadow Philosophy

The Hermès system is intentionally shadowless. Depth is communicated exclusively through surface color contrast (parchment → white → darker parchment) and overlay opacity. This is a conscious luxury decision: shadows imply physical weight and manufactured dimension, while Hermès prefers the flat, paper-like quality of a printed lookbook. The only "shadow" in the system is the inset ring on alerts — which functions as a border emphasis, not an elevation cue.

## 7. Do's and Don'ts

### Do's

- ✅ Use Overpass Mono at weight 300, uppercase, zero letter-spacing for all section headings — this is the Hermès digital signature
- ✅ Use EB Garamond italic at weight 500 for editorial/storytelling display text — the voice of Hermès craftsmanship
- ✅ Set the body background to `#fcf7f1` (Hermès Ivory) — never pure white, never gray
- ✅ Use pure black `#000000` for all buttons, borders, text, and interactive elements — black is the only UI color
- ✅ Apply `text-transform: uppercase` to all heading levels (H1–H5) and labels
- ✅ Use 1px letter-spacing only for navigation labels and CTA labels — spacing signals interactivity
- ✅ Use `border-bottom: 1px solid` for underline links instead of `text-decoration` on editorial backgrounds
- ✅ Set base font-size to 75% on the body — this 12px base is fundamental to all rem calculations

### Don'ts

- ❌ Don't use Hermès Orange (`#f6811e`) as a CTA color, link color, or UI accent — orange is brand-mark-only on digital
- ❌ Don't add box-shadows or elevation to surfaces — the system is intentionally flat, using surface color contrast for hierarchy
- ❌ Don't use backdrop-filter or glassmorphism — the header uses semi-transparent warm tones (`#fcf7f1ba`), not blur
- ❌ Don't use weight 600–800 for headings — Hermès headings live at weight 300 (Overpass Mono) or 500 (EB Garamond)
- ❌ Don't use Manrope for headings — Manrope is the body/UI font; headings belong to Overpass Mono or EB Garamond
- ❌ Don't use positive letter-spacing on body text or headings — letter-spacing 0 is the system default; only nav/CTA labels get 1px
- ❌ Don't use pill-shaped buttons (border-radius ≥ 50px) — the maximum button radius is 4px
- ❌ Don't use cool grays (#f0f0f0, #e5e5e5) for backgrounds — all Hermès neutrals are warm-tinted (#fcf7f1, #f6f1eb, #fffcf7)
- ❌ Don't use title case or sentence case for section headings — all headings are uppercase
- ❌ Don't add a visible bottom border to the header navigation — the header sits directly on the parchment without a dividing line
- ❌ Don't use Filosofia, Akkurat, or Jungle fonts without explicit seasonal campaign authorization — they're declared but not used in the base system

## 8. Responsive Behavior

### Breakpoints

| Name | Width | Key Changes |
|------|-------|-------------|
| Mobile | < 1024px | Header height 50px, side padding 15px, product grid 2-col |
| Desktop | ≥ 1024px | Header height 64px, container max-width 1290px, search input visible (129px), button max-width 320px, product grid 4-col |

### Touch Targets

- Button minimum dimensions: 200px wide × ~42px tall (11px padding × 2 + 20px line-height)
- Navigation links: padded with `0 12px` or `0 15px` for comfortable tap targets
- The system relies on generous padding rather than explicit min-height/min-width constraints

### Collapsing Strategy

- Header shrinks from 64px to 50px on mobile; the category sub-navigation collapses into a hamburger menu
- The search input transitions from a persistent 129px field on desktop to an expandable overlay on mobile
- Typography does NOT downscale significantly — the base font-size (75% / 12px) and the rem-based scale remain consistent across breakpoints
- CJK locales (Japanese, Korean, Chinese) have dedicated font stacks and slightly reduced heading sizes declared via `:root[lang^=ja]` / `[lang^=ko]` / `[lang^=zh-Hans]` / `[lang^=zh-Hant]`

### Image Behavior

- Product and editorial images are delivered through an Adobe Scene7 image server (assets.hermes.com/is/image/hermesedito/)
- Lazy loading with base64 transparent GIF placeholders (`data:image/gif;base64,...`)
- Images fill their containers without explicit aspect-ratio constraints — the layout accommodates varying proportions

### Iteration Guide

1. Always set the page background to `#fcf7f1` — pure white backgrounds are wrong for Hermès; the warmth is non-negotiable
2. Use Overpass Mono weight 300 uppercase for ALL heading levels — switching to Manrope or adding weight destroys the gallery-catalogue character
3. EB Garamond italic is a storytelling tool, not a layout tool — reserve it for hero display text and editorial immersive sections only
4. Button radius is 4px maximum — any radius above 4px (rounded corners, pills) breaks the Hermès visual language
5. The only shadows in the system are inset rings on alerts (`inset 0 0 6px 6px #000`) — do not add drop shadows or elevation to any surface
6. Letter-spacing 0 is the default EVERYWHERE except navigation (1px) and CTA labels (1px) — positive tracking on body text feels like a different brand
7. Body font-size is 75% (12px base) — do not reset to 100% (16px) or all rem-based sizing breaks; 0.75rem = 9px, 1rem = 12px in this system
8. Transitions use `cubic-bezier(.25,.8,.25,1)` for opacity/overlay animations and `.3s` for structural transforms — this easing is softer than standard ease-in-out
