## 1. Visual Theme & Atmosphere

CHANEL's digital presence is an exercise in disciplined restraint. The palette is nearly achromatic — a tightly controlled spectrum from pure black through a series of calibrated grays to white — with absolutely no chromatic accent color. Where most luxury brands might reach for gold or deep navy to signal opulence, CHANEL leans on the absence of color itself. Black is the brand. The interplay of `#000000`, `#1d1d1d`, and `#ffffff` creates all the hierarchy the system needs, letting product photography carry every ounce of chromatic intensity.

The type system is built on two proprietary families: **abchanel-2022** for display-scale headlines (weight 200, an ethereal extra-light that gives headings a whispered authority) and **abchanel-corpo** for all body and UI text (weights 200 through 600, with the default resting at 300). A serif counterpoint, **AWConqueror Pro Didot**, appears in editorial fashion contexts — a modern Didot that bridges the house's print heritage with its digital expression. The result is a typographic palette that feels curated rather than constructed: lightweight, airy, and unmistakably haute couture.

Surfaces are deliberately flat. Shadows are functionally absent — the system explicitly declares `box-shadow: none` across most components. Depth comes from border contrast and the subtle step between `#ffffff` surfaces and `#f9f9f9` variant backgrounds. This is a conscious design choice: the brand trusts its photography and negative space to create visual interest, not layered elevation or glassmorphism.

Every corner is sharp. Buttons have `border-radius: 0`. Cards have no rounding. Inputs are borderless lines, not contained fields. This is the single most distinctive visual choice — the refusal to round anything. In a web ecosystem drowning in 8px radius and pill shapes, CHANEL's zero-radius posture is a statement of identity as powerful as any color.

**Key Characteristics:**
- Pure black-and-white palette with zero chromatic accent — photography provides all color
- Weight 200 (extra-light) for display headings; weight 300 (light) as the body default
- Sharp zero-radius corners on every interactive element — buttons, cards, inputs
- Flat design with no elevation or box-shadow — depth via surface-color steps only
- Two proprietary sans-serif families (abchanel-2022, abchanel-corpo) plus Didot serif for editorial
- Extreme typographic lightness: most text lives at weight 300, with 600 reserved for interactive elements
- Generous whitespace proportions driven by percentage-based grid margins
- Understated text-decoration underline transitions (0.2s ease) as the primary hover signal
- Photography-first: the UI withdraws to let imagery command attention
- Multi-script support (Arabic, Thai, Vietnamese, Korean, Japanese, Chinese) with dedicated font families

## 2. Color Palette & Roles

### Primary

- **CHANEL Black** (`#000000`): `--color-primary`. The brand identity color — logo, navigation text, primary CTA backgrounds, and immersive section fills. Used sparingly but definitively.

### Brand & Dark

- **Near Black** (`#1d1d1d`): `--color-primary-variant`. The most-used text color across the system (105 references). Slightly softer than pure black for extended reading, while remaining nearly indistinguishable in small contexts.
- **Dark Gray** (`#333333`): `--color-secondary`. Used for secondary text hierarchies and body copy in specific contexts.

### Interactive

- **Black CTA** (`#000000`): Primary button background. Hover transitions to `#1d1d1d` or `#f1f1f1` depending on button variant.
- **Disabled Gray** (`#949494`): `--color-primary-disable`. Applied to disabled buttons, inputs, and inactive interactive states.
- **Underline Transition**: Interactive text elements use `text-decoration-color` transitions at 0.2s ease — no background-color shifts on links.

### Neutral Scale

- **Near Black** (`#1d1d1d`): Primary text and headings — `--color-primary-variant`
- **Body Element** (`#454545`): `--color-new-body-element`. Secondary body text in product contexts.
- **Muted Gray** (`#767676`): `--color-secondary-light`. Labels, captions, placeholder text, timestamps.
- **Mid Gray** (`#949494`): `--color-primary-disable`. Disabled state text.
- **Strong Gray** (`#b6b6b6`): `--color-tertiary-strong`. Subtle decorative borders.

### Surface & Borders

- **Pure White** (`#ffffff`): `--color-background`. Default page background.
- **Warm White** (`#f9f9f9`): `--color-background-variant`. Card backgrounds, alternate sections, secondary surfaces.
- **Extra Light Gray** (`#f1f1f1`): `--color-tertiary-extra-light`. Hover state backgrounds.
- **Light Gray** (`#ececec`): `--color-tertiary-light`. Thin separators, subtle dividers.
- **Medium Gray** (`#d8d8d8`): `--color-tertiary-medium`. Default borders on inputs and structural lines.

### Error

- **CHANEL Red** (`#cd0000`): `--color-error`. Validation errors, mandatory field indicators, destructive actions.

### Color Philosophy

The achromatic palette is not a limitation — it is the brand's most recognizable design decision. By removing color entirely from the UI layer, CHANEL creates a gallery-like environment where product photography becomes the sole source of vibrancy. The gray scale is uncommonly fine-grained (7+ distinct values from `#1d1d1d` to `#f9f9f9`), enabling nuanced hierarchy without ever introducing hue. The only non-gray in the system is the error red `#cd0000`, and even that is a deep, restrained crimson rather than a bright alert. This is a system that treats color the way a black-and-white photographer treats light: every tonal value is deliberate, every step intentional.

## 3. Typography Rules

### Font Family

**Primary Display**: `abchanel-2022` — a proprietary geometric sans-serif used exclusively for hero headings and large display text. Available in weights 200 (ExtraLight), 300 (Light), 400 (Regular), and 600 (SemiBold), with italic variants. The brand's typographic fingerprint at scale.

**Primary Body**: `abchanel-corpo` — a proprietary sans-serif for all body copy, navigation, buttons, labels, and UI elements. Available in weights 200, 300, 400, and 600. The workhorse of the system — every piece of functional text uses this face.

**Editorial Serif**: `awconqueror-prodido` (AWConqueror Pro Didot) — a modern Didot serif used in fashion editorial contexts. Available in weights 400, 600, 700, and 800. Bridges print couture heritage with digital editorial.

**UI Complement**: `futura-pt` — Futura PT appears as a supplementary sans-serif for specific UI contexts. Weights 400 and 700 with italic variants.

**Arabic**: Cairo, Tajawal — dedicated Arabic-script families for RTL markets.
**Thai**: Prompt — Thai-script family, weights 300–600.
**Localized Corpo**: `abchanel-corpo-turkey`, `abchanel-corpo-vietnam` — localized versions for Turkish and Vietnamese character coverage.

No monospace font is declared in the system.

- Weight 300 (Light) is the default for all body text in `abchanel-corpo` — this unusual lightness creates the distinctive CHANEL airiness.
- Weight 600 (SemiBold) is the functional weight — buttons, navigation links, and interactive labels use it to create contrast against the 300-weight body without reaching for Bold (700).
- Letter-spacing is near-zero (`0.00625rem` ≈ 0.1px) across most roles — the system does not use tracking as a hierarchy tool.
- The `abchanel-2022` / `abchanel-corpo` split is intentional: display faces use a different optical design than body faces, similar to how print typography distinguishes display cuts from text cuts.
- The serif `awconqueror-prodido` (Didot) appears only in fashion editorial contexts and should never be used for UI elements, navigation, or functional text.

## 5. Layout Principles

### Spacing System

Base unit is not a strict mathematical progression. The system uses a custom scale rooted in `rem` values:
- 0.3125rem (5px), 0.5625rem (9px), 0.875rem (14px), 1rem (16px), 1.125rem (18px), 1.6875rem (27px), 2.25rem (36px), 2.8125rem (45px), 3.375rem (54px), 6.75rem (108px)

This is a non-standard scale that avoids the typical 4px or 8px base. Values like 27px and 45px suggest a base-9 or base-18 system with deliberate intermediary steps.

### Grid & Container

The grid system is declared through CSS custom properties and adapts across 9 breakpoints:
- Mobile (≤320px): 6 columns, 5.625% margins, 16px gap
- Mobile (321px+): 6 columns, 5.333% margins, 19px gap
- Mobile (376px+): 6 columns, 5.5% margins, 18px gap
- Tablet (601px+): 12 columns, 8.095% margins, 16px gap
- Tablet (841px+): 12 columns, 8.021% margins, 19px gap
- Desktop (961px+): 12 columns, 7.917% margins, 22px gap
- Large (1201px+): 12 columns, 8.056% margins, 28px gap
- XL (1441px+): 12 columns, 8.188% margins, 30px gap
- XXL (1601px+): 12 columns, 8.1875rem fixed margins

No max-width constraint is declared — the grid uses percentage-based margins to scale with viewport width, with a fixed margin (131px) kicking in at 1601px+.

### Whitespace Philosophy

CHANEL uses whitespace as its primary luxury signal. Section padding values of 45px (2.8125rem), 54px (3.375rem), and even 108px (6.75rem) create immense breathing room between content blocks. This is not empty space — it is the visual equivalent of silence in a luxury boutique. The percentage-based grid margins (consistently ~8% on desktop) ensure the content never feels edge-to-edge, maintaining the sense of curated restraint regardless of screen size.

### Border Radius Scale

- **0px** — Default for all interactive elements (buttons, inputs, cards). The dominant radius.
- **4px (.25rem)** — Appears on minor UI elements only.
- **6px (.375rem)** — Minor surface rounding (rare).
- **12px (.75rem)** — Chat/overlay-specific contexts, search input.
- **50% / 100%** — Circular elements only (color swatches, radio buttons, avatar placeholders).

The system intentionally avoids the 8px–16px range where most modern design systems live. This is a deliberate brand decision.

## 6. Depth & Elevation

| Level | Treatment | Use |
|-------|-----------|-----|
| Flat (default) | No shadow, no elevation | All standard surfaces, cards, buttons |
| Ambient | `0 1px 7px 0 rgba(0,0,0,0.08)` | Rare — select floating panels |
| Standard | `0 0 18px rgba(0,0,0,0.1)` | Tooltips, dropdowns |
| Elevated | `0 5px 15px 0 rgba(0,0,0,0.08)` | Slide-out panels |
| Focus Ring | `0 0 0 2px #fff, 0 0 0 4px #000` | Focus-visible accessibility ring |

### Shadow Philosophy

The system is intentionally low-elevation. CHANEL explicitly resets `box-shadow: none !important` on most button and interactive selectors, overriding any inherited shadows. Depth is communicated almost exclusively through surface-color contrast — the one-step shift from `#ffffff` to `#f9f9f9` does the work that shadows would do in other systems. When shadows do appear (tooltips, overlays), they are achromatic and subtle (`rgba(0,0,0,0.08–0.1)`), never tinted. This reinforces the gallery-wall aesthetic: nothing projects forward, everything exists on a single plane.

## 7. Do's and Don'ts

### Do's

- ✅ Use `abchanel-2022` at weight 200 (ExtraLight) for hero display headings — this is the brand's typographic signature
- ✅ Use weight 300 (Light) for body text in `abchanel-corpo` — the default lightness is intentional
- ✅ Use weight 600 (SemiBold) for buttons and navigation labels — the only "bold" weight in active UI
- ✅ Keep all interactive elements at `border-radius: 0` — sharp corners are CHANEL's strongest geometric signal
- ✅ Use the `#ffffff` → `#f9f9f9` surface step to create visual hierarchy between sections
- ✅ Let product photography carry all chromatic intensity — the UI must remain achromatic
- ✅ Use `text-decoration-color` transitions at `0.2s ease` for link/button hover states
- ✅ Apply the double-ring focus pattern (`0 0 0 2px #fff, 0 0 0 4px #000`) for accessibility
- ✅ Use `awconqueror-prodido` (Didot) for fashion editorial display moments
- ✅ Maintain generous section spacing (45px–108px) to preserve the luxury silence

### Don'ts

- ❌ Don't round buttons, cards, or inputs — `border-radius: 0` is the CHANEL geometry rule, not a default to override
- ❌ Don't use any chromatic accent color (blue, purple, teal) for CTAs — CHANEL buttons are black or light gray, never colored
- ❌ Don't use `abchanel-2022` below weight 300 for body text — it is designed for display scale only
- ❌ Don't use weight 700 (Bold) or 800 (ExtraBold) for headings — CHANEL headings are light (200–300), never heavy
- ❌ Don't add box-shadows to cards or buttons — the system explicitly resets them with `!important`
- ❌ Don't add backdrop-blur or glassmorphism effects — the visual language relies on solid, opaque surfaces
- ❌ Don't use `futura-pt` for customer-facing text — it is a supplementary face for specific internal or legacy contexts
- ❌ Don't introduce pill shapes (`border-radius: 9999px`) for buttons or badges — zero-radius is absolute
- ❌ Don't use letter-spacing as a hierarchy tool — the system uses near-zero tracking (`0.00625rem`) uniformly
- ❌ Don't apply gradient backgrounds or gradient text effects — the palette is strictly flat achromatic
- ❌ Don't use font-weight 600 for body text — it is reserved for interactive elements (buttons, links, labels)
