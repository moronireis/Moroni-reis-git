Hoje as telas mostram dados de demonstração. Para trocar por dados reais, precisamos conectar duas coisas: WhatsApp e Corp.

2. Conexão WhatsApp — próximo passo

Vamos usar a Evolution API (a mesma tecnologia que já usamos em outros projetos) para conectar os números de WhatsApp da Marpe à plataforma. Funciona 24 horas, sem depender de PC ligado.

O que precisamos de você, Marcel:

Vamos criar uma instância para a Marpe e gerar um QR Code. Você escaneia com o WhatsApp de cada número (como faz com o WhatsApp Web) e pronto — as conversas começam a aparecer na plataforma em tempo real.

Como vai funcionar:

1

Nós geramos o QR Code

Criamos a instância no servidor e te mandamos o QR Code por aqui ou em uma call rápida.

2

Você escaneia com o WhatsApp

Igual WhatsApp Web — abre o WhatsApp no celular, vai em “Aparelhos conectados” e escaneia. Leva 10 segundos por número.

3

Conversas aparecem na plataforma

A partir desse momento, todas as mensagens recebidas e enviadas ficam salvas no CRM. Histórico permanente, sem depender de PC ligado.

Quantos números vamos conectar agora? Na última reunião combinamos 3 (Marcel, Vanessa, Adria). Podemos começar com 1 para testar e depois adicionar os outros.

Só precisamos das credenciais corretas para conectar. A API usa autenticação por e-mail e senha via POST /login.

O que precisamos de você, Marcel:

1

Credenciais da API Corp

Precisamos de um e-mail e senha que tenha acesso à API do Corp (Corp+). Se não tiver certeza se a sua conta tem esse acesso, entre em contato com o suporte da Agia/Agger e pergunte sobre o módulo Corp+ (API REST) — pode ser que precise de ativação.

2

Código da filial (codfil)

A maioria dos endpoints usa um parâmetro codfil. Provavelmente é 1, mas confirme com a Agia se tiver dúvida.

Com essas credenciais, a gente puxa automaticamente toda a carteira de ~1.000 clientes e 1.362 contratos para dentro do CRM.
