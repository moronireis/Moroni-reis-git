2. Estratégia WhatsApp

Marcel levantou um ponto crítico na reunião de 12/05: a API oficial Meta exige templates aprovados para iniciar conversa, o que não serve para o fluxo diário receptivo da Marpe.

Decisão recomendada: modelo híbrido

Usar WhatsApp Web (não oficial) para o inbox diário (receptivo, conversas livres, grupos) + API oficial Meta para automações específicas (boas-vindas, aviso de parcela, pesquisa de satisfação — que são templates prontos). Isso evita a limitação de templates no atendimento do dia a dia.

CanalUsoVantagem
WhatsApp Web (3 números)Inbox diário, conversas livres, grupos de seguradoras, atendimento receptivoSem restrição de templates. Histórico salvo. Funciona como WaSpeed mas 24/7.
API oficial Meta (1 número dedicado)Automações: boas-vindas, aviso parcela, pesquisa satisfação, campanhas em massa24/7 sem PC. Templates aprovados. Ideal para disparos automáticos.

4. Processos Mapeados (Reunião 12/05)

4.1 Fluxo de Vendas (Negociação)

  1. Cadastrar cliente no Corp (ou buscar existente). Campos: nome, CPF/CNPJ, endereço, telefone, e-mail, profissão, estado civil, dados bancários.
  2. Criar negociação na Central de Negócios do Corp. Campos: cliente, tipo (prospecção/endosso), ramo, campanha, dados do seguro atual (seguradora, vigência, corretora anterior), estimativa de prêmio, comissão, produtor, data de aproximação (controle principal), status.
  3. Cotar no Agilizador (sistema separado, sem integração com Corp). Gerar orçamento PDF.
  4. Enviar cotação ao cliente via WhatsApp. Registrar atendimento no Corp + anexar PDF.
  5. Negociar via WhatsApp. Atualizar status + próxima ação no Corp.
  6. Fechar: registrar “cliente de acordo”, mudar responsável para Adria.
  7. Adria transmite proposta na seguradora. Cadastra contrato no Corp via InCorp (importação automática de PDF).
  8. Acompanhar emissão: pendente → emitido → recebido → entregue (3 estágios).
  9. Boas-vindas: Adria envia mensagem padrão + números de assistência.

4.2 Fluxo de Renovação

  1. Todo dia 20 do mês: Marcel consulta Oportunidades → Renovações no Corp (contratos com vigência vencendo no mês seguinte).
  2. Seleciona todos e clica “Gerar Negociações” → cria cards no CRM com status “iniciar negociação para renovação”.
  3. Fluxo segue igual ao de vendas (contato → cotação → fechamento).

Oportunidade de automação: gerar negociações de renovação automaticamente (sync Corp) + disparar primeiro contato via WhatsApp sem intervenção manual.

4.3 Fluxo de Resgate + Venda Cruzada

  • Resgates: contratos não renovados no ano anterior. Corp lista automaticamente. Marcel decide resgatar ou descartar.
  • Venda cruzada: filtros por ramo + perfil demográfico (estado civil, profissão, idade, sexo). Ex: oferecer auto para quem tem vida. Gera negociações em lote.
  • Problema atual: campo “motivo de perda” não é obrigatório no Corp → sem dados de por que perdeu. No novo CRM, tornar obrigatório com opções configuráveis.

4.4 Fluxo de Sinistro / Assistência 24h

  1. Abertura: cadastrar no Corp (data, hora, protocolo, tipo, item segurado, franquia, descrição).
  2. Acompanhamento: registrar histórico (vistoria, autorização, oficina, status). 4 status: pendente, aberto, em andamento, concluído.
  3. Automações por mudança de status:
    • Aberto → msg “atendimento aberto, protocolo #X”
    • Prestador acionado → msg “prestador a caminho, previsão X min”
    • Prestador chegou → msg confirmação
    • Concluído → pesquisa de satisfação
  4. Variáveis nos templates: #primeiro_nome, #protocolo, #veículo, #oficina, #data_evento, #franquia, #período_dia
