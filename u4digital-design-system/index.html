Shadows

Structural, not theatrical. Shadows separate surfaces — they do not decorate them. Never use drop shadows for emphasis; use borders and surface contrast instead.

Quick Start

Two files. No build tools. Drop both into your project and start building.

HTML Setup
<!-- 1. Inter from Google Fonts --> <link rel="preconnect" href="https://fonts.googleapis.com" /> <link href="https://fonts.googleapis.com/css2?family=Inter:wght@300;400;500;600;700&display=swap" rel="stylesheet" /> <!-- 2. Design system — ORDER MATTERS: tokens first, then components --> <link rel="stylesheet" href="design-system.css" /> <link rel="stylesheet" href="components.css" />
Using Tokens in Your CSS
/* Use tokens — never magic numbers */ .my-component { background: var(--color-bg-card); border: 1px solid var(--color-border); border-radius: var(--radius-lg); padding: var(--space-6); font-size: var(--font-sm); color: var(--color-text-secondary); transition: border-color var(--transition-fast); } .my-component:hover { border-color: var(--color-border-strong); } /* Accent green = signal, not decoration */ .my-component--active { border-left: 2px solid var(--color-accent); background: var(--color-accent-muted); }
Component Composition
<!-- Primary action with icon --> <button class="btn btn--primary">Deploy System</button> <!-- Status badge --> <span class="badge badge--success"> <span class="badge__dot"></span> Running </span> <!-- Interactive card --> <div class="card card--interactive card--accent"> <div class="card__header"> <h3 class="card__title">Lead Qualifier</h3> <p class="card__subtitle">AI / ML</p> </div> <p class="card__body">Qualifies leads from 5 sources in real time.</p> </div>
Green Usage Rule

Green (#74C69E) is a signal color — use it for one thing per screen: the primary CTA, an active/success state, or a single data point that needs attention. When everything is highlighted, nothing is. Never use green for decoration, background fills, or as a dominant hue.
